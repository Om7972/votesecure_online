System Architecture

Understanding how VoteSecure protects voter privacy while ensuring integrity.

Client-Side Encryption

Votes are encrypted in the user's browser. The server never sees the raw vote choice.

Blind Signature Service

The server verifies eligibility and signs the encrypted vote without knowing its contents.

Blockchain Ledger

The signed vote is submitted to the immutable ledger. Results are calculated via homomorphic tallying.

Creating Elections

Learn how to set up highly configurable elections with diverse ballot types.

Step-by-Step Configuration

  1. Define Ballot Type: Choose between First-Past-The-Post, Ranked Choice (RCV), or Approval Voting.
  2. Set Timelines: Configure precise start and end times (UTC) for the voting window.
  3. Add Candidates: Upload candidate profiles, including bios, photos, and party affiliations.
  4. Configure Security: Enable/disable strict IP checking, biometric requirements, and result visibility.

Pro Tip

Use 'Test Mode' to simulate the election flow with dummy voters before going live.
