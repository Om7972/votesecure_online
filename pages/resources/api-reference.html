API Reference

Integrate VoteSecure functionality directly into your applications.

Authentication

All API requests must be authenticated using a Bearer Token.

Users

Manage user accounts and profile information.

GET

/api/users/me

Retrieve the authenticated user's profile information. 

{
  "success": true,
  "user": {
    "id": 1,
    "name": "Jane User",
    "email": "jane@example.com",
    "role": "voter"
  }
}
GET

/api/elections/:id

Get detailed information about a specific election, including candidates.

{
  "success": true,
  "election": {
    "id": 101,
    "title": "City Council 2026",
    "description": "Vote for your local representative.",
    "candidates": [
        { "id": 1, "name": "Alice Voter", "party": "Progressive" },
        { "id": 2, "name": "Bob Ballot", "party": "Conservative" }
    ]
  }
}
GET

/api/elections

Retrieve a list of active and upcoming elections.

Parameters

  • status Optional. 'active' | 'upcoming'
Response 200 OK 
{
  "success": true,
  "data": [
    {
      "id": 101,
      "title": "City Council 2026",
      "status": "active",
      "end_date": "2026-11-04T00:00:00Z"
    }
  ]
}
POST

/api/elections/:id/vote

Submit a ballot for a specific election.

Body

  • candidate_id Required. Integer
Response 200 OK 
{
  "success": true,
  "message": "Vote cast successfully",
  "receipt_hash": "0x8f2a...c4b1"
}
